# Phase 0 · 入口分流与需求确认

## 路径判断

- 明确人名/主题：直接进入蒸馏
- 模糊需求：先做1-2轮诊断，再给2-3个候选

## 直接路径最小确认项

1. 蒸馏对象是否正确
2. 目标用途：思维顾问 / 决策参考 / 角色扮演
3. 新建还是更新
4. 是否有本地一手素材

## 诊断路径

- 先判断是人物Skill还是主题Skill
- 候选输出必须包含：核心镜片、适配理由、局限
- 不超过3个候选
